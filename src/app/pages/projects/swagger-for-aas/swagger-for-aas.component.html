Swagger 4 AAS

Automating API Documentation for Asset Administration Shells

An Asset Administration Shell (AAS) is a standardised digital representation of a physical or virtual asset, enabling interoperability in Industry 4.0. It encapsulates all relevant data, functionalities and lifecycle information, acting as a digital twin for seamless integration and automation. Through structured submodels, AAS facilitates efficient data exchange, traceability and smart decision-making. Swagger UI is an open-source tool that visually represents APIs, allowing developers to explore, test, and interact with API endpoints through an intuitive web interface. It generates documentation from OpenAPI specifications, making it easier to understand request and response structures, parameters, and authentication requirements.

The Asset Administration Shell (AAS) is stored in a .aasx file, which serves as a standardized container for managing and exchanging digital twin data. These files can be opened, viewed, and edited using tools such as the AASX Package Explorer, allowing users to inspect and modify the contained information efficiently.

The Swagger UI requires a predefined format, which is illustrated in a tree diagram. This diagram provides a structured overview of how the data should be organized. The information from an Asset Administration Shell must ultimately be parsed into this format to ensure compatibility and proper integration.

The implementation is done in Java, where all necessary data is read from an Asset Administration Shell via REST. While there are variations in the standard, companies typically maintain a consistent internal structure, allowing a Swagger UI to be generated for a specific format. Predefined classes are used to represent the required Swagger UI format. These classes are populated with the extracted information from the Asset Administration Shell. Finally, instances of these classes are converted into a JSON string, which serves as the foundation for the Swagger UI.
